Overview

simple_codegen is a CLI tool that guides Claude (AI) through a strict, lock-file controlled pipeline for generating Eiffel code. Each step requires explicit permission via the lock file, preventing the AI from skipping steps or batching operations.

Part of the simple_* ecosystem of focused, single-purpose Eiffel libraries with full Design by Contract support.

Quick Start

Installation

Set environment variable and build:

-- Set SIMPLE_EIFFEL=D:\prod
cd $SIMPLE_EIFFEL/simple_code
/d/prod/ec.sh test -config simple_code.ecf -target simple_codegen

Initialize a Session

# Initialize a new session
simple_codegen init --session my_project

# Generate specification
simple_codegen process --session my_project

# Generate classes (one at a time via lock file)
simple_codegen process --session my_project

# Compile and fix errors (one at a time)
simple_codegen compile --session my_project --project /path/to/project

# Run tests
simple_codegen run-tests --session my_project --project /path/to/project

Features

Lock-File Pipeline

Every action requires lock file permission - no skipping or batching.

Session Management

Track progress across multiple sessions with persistent state.

TOON Prompts

AI-readable prompts for consistent code generation.

Compile Integration

Run ec.exe and parse error output automatically.

Reuse Discovery

Find existing simple_* code for reuse in new projects.

Security Analysis

Scan generated code for potential vulnerabilities.

CLI Reference

Commands

CommandDescription
init --session NAMEInitialize new code generation session
process --session NAMEProcess next step in pipeline
compile --session NAME --project PATHCompile project and track errors
compile --fix --session NAMEFix ONE compile error
run-tests --session NAME --project PATHRun tests and track failures
run-tests --fix --session NAMEFix ONE test failure
lock --session NAMEShow current lock file state
